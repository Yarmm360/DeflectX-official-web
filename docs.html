DeflectX — a desktop SIEM that watches your logs and tells you what matters

This page covers everything from first launch to writing your own detection rules — whether you're running the Python script directly or a packaged Windows .exe.

§Overview

What DeflectX watches, and what it does when it doesn't like what it sees.

DeflectX tails one or more system log sources in real time, checks every line against a set of regex rules, and raises an alert the moment something suspicious shows up — a failed login, a new user account, a cleared audit log, a service quietly installing itself. Alerts land in a live table, get written to a text log and a SQLite database, and can trigger a tray notification or a beep.

Multi-source
Watch journalctl, the Windows Event Log, a tailed file, or any custom command — all at once.
Editable rules
Add, edit, or remove detection rules from the app. No code changes needed.
GeoIP enrichment
Public IPs found in a matching line get a country and city, looked up automatically.
History & charts
Every alert is stored in SQLite. The Analytics tab charts alerts by rule and over time.

§Install & run

Get the app running from source in under a minute.

  1. Install PyQt6 — the only hard requirement. 
    pip install PyQt6
  2. (Optional) Install the extras — matplotlib powers the Analytics tab; pywin32 is needed only on Windows for the Event Log source. 
    pip install matplotlib pywin32
  3. Run it. 
    python siem_gui.py
First run A siem_config.json file is created automatically next to the script, with sensible defaults for your operating system — journalctl on Linux, the Windows System event log on Windows. Nothing to set up before your first launch.
Common snag If you see ModuleNotFoundError: No module named 'PyQt6', it usually means pip install and python point at two different interpreters. Check which interpreter your editor or terminal is actually using and install into that one.

§Build a Windows .exe WINDOWS

Package the app into a single file that runs without a Python install.

This uses PyInstaller, and it has to run on Windows — PyInstaller doesn't cross-compile from another OS.

  1. Install the build tools: 
    pip install pyinstaller PyQt6 pywin32 matplotlib
  2. From the folder containing siem_gui.py, run: 
    pyinstaller --noconfirm --onefile --windowed --name "DeflectX" --hidden-import win32timezone --hidden-import win32evtlog --hidden-import win32evtlogutil --collect-submodules matplotlib.backends siem_gui.py
  3. Find your executable at dist\DeflectX.exe — it's fully self-contained.
FlagWhat it does
--onefileBundles everything into one .exe (slightly slower to start; unpacks to a temp folder each launch)
--windowedHides the console window, since this is a GUI app
--hidden-import win32timezoneFixes a timestamp bug PyInstaller commonly misses for Event Log sources
Where files land The packaged app saves its config, database, and text log next to DeflectX.exe itself — not wherever the current working directory happens to be when it's launched.

§Live Dashboard tour

Everything on the main screen, and what it's for.

Toolbar

  • Start Monitoring — begins tailing every enabled source. The status dot turns green while running.
  • Stop — stops all sources. Existing alerts stay in the table.
  • Clear Alerts View — empties the table on screen only. Nothing is deleted from the text log or database.

Filter row

Narrow the alert table by rule, severity, or a free-text search that matches across the rule name, source, IP, location, and log text.

Alerts table

Each row shows the timestamp, which source it came from, the rule that matched, severity, any public IP found in the line, its resolved location, and the raw log text — color-coded by severity so the worst alerts are the easiest to spot while scanning.

Live feed panel

Below the table, every line from every active source scrolls by in real time, tagged with its source name — useful for confirming a source is actually producing data, even when nothing has triggered an alert yet.

§Severity levels

Every rule is tagged with one of three severities, used for row color, filtering, and notification thresholds.

Critical Warning Info

Critical is for things that are rarely benign — an audit log getting cleared, a new admin-group member, a deleted account. Warning covers things worth a look but often legitimate — a failed login, a new service. Info is background context, like a routine sudo command or a new process spawning.

§Analytics tab

A second view next to the dashboard, built from everything stored in SQLite.

Two charts: alerts grouped by rule, and alerts per minute over time, so a spike is obvious at a glance. Press Refresh Charts to pull the latest data. This tab needs matplotlib installed — if it isn't, you'll see an install hint here instead of a broken screen.

§Storage & export

Where your alert history actually lives.

FileContains

Use File → Export Alerts to CSV to save whatever's currently visible in the table (filters apply). File → Show Storage File Paths prints the exact location of all three files.

§Detection rules

Manage → Detection Rules opens an editable table — no code required.

Each rule has a name, a regex pattern checked against every incoming line, a severity, and an optional ignore case flag. The first rule that matches a line wins — order doesn't otherwise matter.

Rule changes apply immediately to anything already running — no need to stop and restart monitoring. Log sources work differently; see below.

Writing patterns Keep patterns specific enough to avoid false positives, but not so specific they only match one exact log line. Test a new rule by watching the live feed for a line you expect it to catch, and confirming it lands in the alert table.

§Log sources

Manage → Log Sources controls what DeflectX actually watches.

TypePlatformTarget field
journalctlLINUXNot needed
windows_eventlogWINDOWSLog name — System, Security, or Application
fileAny OSPath to the file to tail
commandAny OSAny command whose output you want streamed

Each source can be individually enabled or disabled. Unlike rules, source changes take effect the next time you click Start — stop monitoring first if it's already running.

Security log access Reading the Windows Security log normally requires running DeflectX as Administrator. The System log doesn't have that restriction.

§GeoIP lookups

Turning a bare IP address into a country and city.

When a matching line contains a public IP address, DeflectX looks it up against the free ip-api.com service and caches the result so the same IP is never looked up twice. Private, loopback, and reserved addresses are skipped automatically.

Privacy note This sends the IP address to a third-party service over the network. Turn it off in Settings if you'd rather keep lookups offline — alerts still work fine, just without a location.

§Notifications

Getting alerted without staring at the window.

A tray icon lets you start, stop, or reopen DeflectX without keeping the window in focus. In Settings, you can turn on desktop popups for alerts at or above a minimum severity, and a beep specifically for critical alerts.

§Windows audit policy WINDOWS

Why the Security log might stay quiet even when it shouldn't.

Windows doesn't log things like failed logons or new user accounts by default. To see those events, turn on Advanced Audit Policy first:

  1. Open secpol.msc → Advanced Audit Policy Configuration, or run auditpol from an elevated prompt.
  2. Enable auditing for the categories you care about — Logon/Logoff and Account Management cover most of DeflectX's default Windows rules.
  3. Add a windows_eventlog source pointed at Security, and run DeflectX as Administrator.

§Troubleshooting

Quick answers to the errors you're most likely to hit.

ModuleNotFoundError: No module named 'PyQt6'

Your terminal and your editor are likely using different Python interpreters. Install PyQt6 into the exact interpreter shown in the error's file path, e.g. /usr/bin/python -m pip install PyQt6.

"journalctl not found"

You're not on a systemd-based Linux distribution. Switch the source type to file or command, or use windows_eventlog on Windows.

No alerts ever appear

Check the live feed panel first — if lines are showing up there but never triggering a rule, your patterns may not match the real log text. On Windows, confirm Advanced Audit Policy is enabled for the events you expect.

Windows Event Log source won't start

Install pywin32 (pip install pywin32). If it's the Security log specifically, relaunch as Administrator.

Analytics tab shows an install message instead of charts

That's expected when matplotlib isn't installed — run pip install matplotlib and reopen the tab.
